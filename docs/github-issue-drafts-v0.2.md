## P0-1 启动后默认显示主页面

**Title**
`[P0] 启动后默认显示设置控制台，取消纯托盘入口`

**Labels**
`P0`, `product`, `desktop`, `frontend`

**Body**
```md
## 背景
当前应用启动后默认进入后台，主窗口不可见。对用户来说，这会造成“应用打开了但看不到”的感知问题，也不利于 v0.2 作为可讨论的产品版本进行试用。

## 目标
将应用入口从“纯托盘原型”调整为“主页面设置控制台优先”。用户双击应用后应立即看到主页面。

## 范围
- 调整 Tauri 窗口配置
- 主窗口默认可见
- 明确启动行为
- 保留托盘能力，但托盘不再是唯一入口

## 非目标
- 不在本 issue 内完成完整设置控制台 UI
- 不在本 issue 内实现首次使用引导

## 交付结果
- 用户启动应用后可直接看到主页面
- 不再出现“启动了但看不见”的情况

## 验收标准
- [ ] 双击应用后 3 秒内看到主页面
- [ ] 托盘仍保留且可正常使用
- [ ] 不再依赖托盘菜单才能进入产品

## 参考资料
- docs/prd-passive-reminder-v0.2.md
- docs/settings-console-ia.md
- docs/issue-backlog-v0.2.md
```
