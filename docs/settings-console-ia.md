# Fragment Vocab 设置控制台 IA

## 1. 文档目的
本文件定义 v0.2 主页面的信息架构，目标是把当前“后台原型”升级为“可理解、可配置、可讨论”的设置控制台。

## 2. 页面定位
主页面不是内容消费页，也不是大屏统计页。

它是一个桌面端设置控制台，承担 4 个职责：
1. 展示当前提醒状态
2. 展示系统推荐策略
3. 提供用户覆盖配置
4. 提供暂停、恢复、诊断与入口管理

## 3. 页面结构
建议采用单窗口、分区块结构，首版不必做复杂多页导航。

### 3.1 顶部区域
- 产品标题
- 当前模式标签
- 当前状态标签
  - 运行中
  - 已暂停至某时间
  - 今日不再提醒
  - 夜间静默中
- 右侧快捷操作
  - 暂停 1 小时
  - 今日不再提醒
  - 恢复提醒

### 3.2 今日状态区
展示内容：
- 今日学习次数
- 认识 / 不认识 / 跳过
- 今日新词数
- 待复习数
- 正确率
- 下次可能提醒时间

目标：
- 让用户先理解“现在产品处于什么状态”

### 3.3 提醒策略区
分为左右两列：

左列：系统推荐
- 当前推荐模式
- 推荐说明
- 推荐参数摘要

右列：用户自定义
- 是否启用自定义
- 自定义后覆盖哪些参数
- 一键恢复系统推荐

包含字段：
- 提醒模式
- idle 阈值
- 兜底频率
- 是否启用兜底

### 3.4 学习计划区
展示和配置：
- 每日新词上限
- 是否优先复习词
- 无复习词时是否引入新词
- 词库来源或当前词库包

目标：
- 将“提醒频率”与“学习强度”分开管理

### 3.5 时间与静默区
展示和配置：
- 夜间静默开始时间
- 夜间静默结束时间
- 工作日策略
- 周末策略
- 当前是否处于静默时段

目标：
- 让用户清楚知道“为什么现在没提醒”

### 3.6 卡片偏好区
展示和配置：
- 自动隐藏时长
- 是否显示音标
- 默认展示顺序
- 是否允许跳过
- 快捷键开关

目标：
- 把提醒节奏和卡片表现解耦

### 3.7 系统设置区
展示和配置：
- 开机启动
- 启动后显示主页面 / 最小化到托盘
- 托盘菜单开关
- 打开统计页

### 3.8 解释与诊断区
展示内容：
- 最近一次提醒时间
- 最近一次未提醒原因
- 当前阻塞条件
  - 暂停中
  - 夜间静默
  - 无可用卡片
  - 卡片已显示中

目标：
- 减少“为什么没反应”的疑惑

## 5. 导航策略
### 5.1 首版
- 单页滚动控制台
- 每个区块可折叠
- 页面顶部保留状态摘要和快捷操作

### 5.2 后续可演进
若设置项继续增长，可拆为左侧导航：
- 总览
- 提醒策略
- 学习计划
- 卡片偏好
- 系统设置
- 诊断

## 6. 关键交互
### 6.1 推荐转自定义
1. 用户看到系统推荐值
2. 点击“切换自定义”
3. 页面提示哪些参数将被覆盖
4. 保存后显示“当前使用自定义”

### 6.2 恢复推荐
1. 用户已自定义
2. 点击“恢复系统推荐”
3. 恢复所有推荐参数
4. 页面显示“当前使用推荐”

### 6.3 暂停与恢复
1. 用户点击“暂停 1 小时”
2. 顶部状态立即变为“已暂停至 xx:xx”
3. 托盘状态同步
4. 到时自动恢复或用户手动恢复

### 6.4 诊断查看
1. 用户觉得提醒太少或太多
2. 打开主页面
3. 在解释与诊断区查看最近一次提醒或阻塞原因

## 7. 数据模型映射建议
为便于研发落地，建议控制台直接映射到以下配置结构：

```ts
interface AppConfig {
  reminder: {
    mode: "gentle" | "balanced" | "intensive" | "custom";
    usingRecommended: boolean;
    idleThresholdSec: number;
    fallbackEnabled: boolean;
    fallbackIntervalMin: number;
  };
  schedule: {
    quietHoursStart: string;
    quietHoursEnd: string;
    weekdayProfile?: string;
    weekendProfile?: string;
  };
  learning: {
    dailyNewLimit: number;
    reviewFirst: boolean;
    allowNewWhenNoDue: boolean;
  };
  card: {
    autoHideSec: number;
    showPhonetic: boolean;
    revealOrder: "en-first" | "zh-first";
    allowSkip: boolean;
    shortcutsEnabled: boolean;
  };
  system: {
    launchAtLogin: boolean;
    startBehavior: "show-main" | "minimize-to-tray";
    trayEnabled: boolean;
  };
}
```

## 8. 关键状态定义
### 8.1 顶部状态标签
- 运行中
- 已暂停
- 今日不再提醒
- 夜间静默
- 当前无卡可学

### 8.2 提醒策略标签
- 系统推荐
- 已自定义
- 推荐已被部分覆盖

## 9. 文案原则
1. 尽量少用技术术语
2. 用户看到的是“提醒频率”，不是“scheduler interval”
3. 尽量解释结果，而不是暴露内部实现
4. 每个关键状态都要有简短说明

## 10. 首版实现边界
首版 IA 不要求：
- 图表化数据分析
- 多层深导航
- 复杂视觉动效
- 团队协作视图

首版只要求：
- 状态清楚
- 配置可改
- 推荐和自定义边界清楚
- 与托盘和浮卡行为一致
