# Fragment Vocab v0.2 开发任务清单

本文件按 `P0 / P1 / P2` 提供可直接建 issue 的任务描述。

## P0

### P0-1 启动后默认显示主页面
- 标题：启动后默认显示设置控制台，取消纯托盘入口
- 目标：应用启动后用户立即看到主页面，而不是后台无感启动
- 范围：
  - 调整 Tauri 窗口配置
  - 主窗口默认可见
  - 明确启动行为
- 验收：
  - 双击应用后能直接看到主页面
  - 不再出现“启动了但看不见”
  - 托盘仍保留

### P0-2 建立设置控制台主页面骨架
- 标题：实现主页面设置控制台首版骨架
- 目标：提供主页面作为产品主入口
- 范围：
  - 顶部状态区
  - 今日状态区
  - 提醒策略区
  - 学习计划区
  - 时间与静默区
  - 卡片偏好区
- 验收：
  - 页面结构与 `docs/settings-console-ia.md` 一致
  - 所有区块有可见占位和基础数据

### P0-3 增加配置模型与持久化
- 标题：为提醒策略和偏好设置建立配置模型与本地持久化
- 目标：用配置驱动产品行为，替代硬编码
- 范围：
  - 设计配置 schema
  - 本地持久化到 SQLite 或配置表
  - 提供读取和更新接口
- 验收：
  - 配置修改后重启仍保留
  - 有默认值回退机制
  - 配置读取失败时不崩溃

### P0-4 调度器改为配置驱动
- 标题：将调度器从硬编码改为读取用户配置
- 目标：idle 阈值、兜底频率、静默时间等都从配置加载
- 范围：
  - 接入配置读取
  - 去除关键硬编码
  - 调整调度器测试
- 验收：
  - 修改提醒配置后行为变化可见
  - 现有调度逻辑单测继续通过

### P0-5 支持系统推荐值与用户覆盖
- 标题：实现推荐提醒策略与自定义覆盖机制
- 目标：系统给出推荐频率，用户可覆盖且可恢复
- 范围：
  - 推荐模式定义
  - 当前模式展示
  - 切换到自定义
  - 恢复推荐
- 验收：
  - 页面能显示“当前使用推荐 / 自定义”
  - 用户修改后立即生效
  - 可一键恢复推荐值

### P0-6 打通暂停状态同步
- 标题：主页面、托盘、调度器之间同步暂停与恢复状态
- 目标：避免各端显示和实际行为不一致
- 范围：
  - 暂停 1 小时
  - 今日不再提醒
  - 恢复提醒
  - 顶部状态同步
- 验收：
  - 从任一入口暂停后，其他入口状态同步更新
  - 到期自动恢复

### P0-7 托盘改为辅助入口
- 标题：重构托盘菜单为辅助操作入口
- 目标：托盘不再承担主入口职责
- 范围：
  - 打开主页面
  - 打开统计页
  - 暂停 1 小时
  - 今日不再提醒
  - 退出应用
- 验收：
  - 托盘功能与主页面功能不冲突
  - 打开主页面是首要入口项

### P0-8 增加提醒诊断信息
- 标题：在主页面展示当前提醒状态和最近阻塞原因
- 目标：减少“为什么没提醒 / 为什么提醒了”的不透明感
- 范围：
  - 最近一次提醒时间
  - 当前阻塞条件
  - 下次可能提醒时间
- 验收：
  - 用户能在主页面看到当前是否被暂停、静默、无卡、卡片占用等原因阻塞

## P1

### P1-1 首次使用引导
- 标题：实现首次使用引导并收集基础偏好
- 目标：降低首次理解成本
- 范围：
  - 每日新词目标
  - 可提醒时间段
  - 偏克制还是偏积极
  - 是否开机启动
- 验收：
  - 首次打开时自动进入引导
  - 完成后生成默认配置

### P1-2 提醒模式预设
- 标题：提供克制 / 平衡 / 强化 / 自定义模式
- 目标：让用户不必从零理解全部参数
- 范围：
  - 模式定义
  - 模式切换
  - 文案说明
- 验收：
  - 模式切换后参数联动更新
  - 默认模式为克制

### P1-3 工作日与周末策略
- 标题：支持工作日与周末不同提醒规则
- 目标：让提醒节奏更贴近日常作息
- 范围：
  - 配置结构扩展
  - 页面设置
  - 调度器判断
- 验收：
  - 不同日期下使用正确策略

### P1-4 卡片偏好设置
- 标题：将卡片展示方式改为可配置
- 目标：让浮卡风格适应不同用户偏好
- 范围：
  - 自动隐藏时间
  - 是否显示音标
  - 默认展示顺序
  - 是否允许跳过
  - 快捷键开关
- 验收：
  - 主页面修改后浮卡行为同步变化

### P1-5 空状态与异常状态设计
- 标题：补齐“无卡可学 / 配置异常 / 读取失败”状态
- 目标：避免用户只看到无反馈或日志输出
- 范围：
  - 主页面空状态
  - 卡片页无卡状态
  - 配置错误提示
- 验收：
  - 无卡可学时主页面有明确说明
  - 不再依赖控制台日志理解状态

### P1-6 推荐说明文案
- 标题：为系统推荐频率增加解释文案
- 目标：建立用户对推荐机制的信任
- 范围：
  - 为什么当前是克制/平衡/强化
  - 如何切到自定义
  - 恢复推荐意味着什么
- 验收：
  - 页面上有清楚说明，不依赖外部文档

## P2

### P2-1 用户反馈入口
- 标题：增加“提醒太多 / 太少 / 不想再看这张词”的反馈入口
- 目标：为后续推荐优化收集信号
- 范围：
  - 主页面反馈入口
  - 浮卡轻反馈
  - 本地记录
- 验收：
  - 至少能记录反馈类型与时间

### P2-2 配置导出与分享
- 标题：支持导出当前配置摘要用于团队讨论
- 目标：支持跨角色评审和试点
- 范围：
  - 导出文本摘要
  - 导出截图或 JSON
- 验收：
  - 团队可复用同一套配置讨论

### P2-3 团队默认模板
- 标题：提供不同团队场景的默认提醒模板
- 目标：适配不同部门的节奏差异
- 范围：
  - 开发 / 运营 / 产品等模板
  - 模板切换
- 验收：
  - 模板能直接应用到配置

### P2-4 推荐策略优化
- 标题：从静态推荐升级到更细粒度的动态推荐
- 目标：让系统推荐更贴合用户接受度
- 范围：
  - 基于跳过率、暂停率、作答率调整推荐
  - 推荐原因回显
- 验收：
  - 推荐逻辑可解释
  - 不影响用户自定义优先级
