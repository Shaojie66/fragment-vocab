# Fragment Vocab 被动提醒版 PRD

## 1. 文档信息
- 版本：v0.2 Draft
- 状态：需求讨论稿
- 更新时间：2026-03-13
- 目标平台：macOS
- 产品形态：主页面设置控制台 + 托盘辅助 + 浮卡提醒

## 2. 产品定位
`Fragment Vocab` 是一个被动提醒型背词工具。

它不要求用户主动进入学习页开始任务，而是在工作间隙、低打扰场景下，由系统推荐合适的提醒频率，用轻量浮卡完成单词记忆闭环。

与现阶段“菜单栏常驻 + 浮卡原型”不同，v0.2 的产品入口应改为主页面设置控制台。用户需要先理解和配置提醒策略，再决定是否接受系统推荐。

## 3. 背景与问题
当前原型已经验证了以下能力：
- 本地词库导入
- SRS 阶段流转
- idle 触发与兜底触发
- 浮卡作答与统计

但当前形态存在产品层问题：
- 启动后没有主页面，用户难以理解产品状态
- 提醒策略几乎全部硬编码，用户无法调优
- 托盘承担了过多主入口职责，不适合团队试用和需求讨论
- 系统推荐与用户自定义之间没有清晰边界

## 4. 产品目标
### 4.1 核心目标
1. 将应用形态从“纯托盘原型”升级为“可配置的被动提醒产品”
2. 让用户清楚知道当前提醒规则、为什么会提醒、何时不会提醒
3. 在默认克制的前提下，提升提醒可接受度而不是追求高频
4. 保留系统推荐能力，但始终给予用户覆盖权

### 4.2 成功标准
1. 用户首次启动后 1 分钟内完成基础提醒设置
2. 用户能在主页面 30 秒内找到并修改提醒频率
3. 用户能明确区分“系统推荐”与“自定义覆盖”
4. 启动后看得到主页面，不再出现“应用打开但不可见”
5. 无卡可学时不会出现空弹窗或误触发

## 5. 非目标
以下内容不纳入本轮需求：
- 多端同步
- 用户账号系统
- AI 个性化学习路径
- 单词详情百科化扩展
- 社交排行榜
- 团队管理后台

## 6. 目标用户
### 6.1 核心用户
- 长时间在电脑前工作的知识工作者
- 希望在碎片时间背词，但不想额外打开学习 App 的用户
- 需要低打扰、可控节奏、可持续使用的学习工具用户

### 6.2 典型特征
- 对打断敏感
- 愿意接受适度自动化
- 希望“提醒有边界、规则可解释”
- 不希望完全把节奏交给系统

## 7. 产品原则
1. 被动提醒，不主动占用主任务流
2. 默认克制，先少打扰，再谈加强
3. 系统给推荐，用户有最终决定权
4. 所有关键状态可见、可解释、可修改
5. 托盘是辅助入口，主页面才是产品主入口

## 8. 核心体验定义
### 8.1 启动体验
1. 用户打开应用
2. 默认看到主页面设置控制台
3. 主页面展示当前学习状态、提醒模式、静默时间、卡片偏好
4. 首次用户进入最小引导，完成基础偏好设置

### 8.2 日常使用体验
1. 系统根据当前模式给出提醒节奏
2. 用户可保留推荐值，也可切换到自定义配置
3. 满足条件时弹出浮卡
4. 用户通过认识 / 不认识 / 跳过完成作答
5. 主页面和托盘都能反映暂停、静默、下次提醒等状态

### 8.3 调整体验
1. 用户感觉打扰太多
2. 打开主页面
3. 直接看到当前策略和推荐理由
4. 一键切到更克制的模式，或手动设置频率

## 9. 功能范围
### 9.1 P0 必须包含
1. 启动显示主页面
2. 主页面设置控制台
3. 系统推荐提醒策略
4. 用户自定义覆盖提醒频率
5. 暂停 / 今日不再提醒 / 恢复提醒
6. 提醒策略可视化说明
7. 托盘辅助入口
8. 浮卡继续作为作答组件

### 9.2 P1 建议包含
1. 首次使用引导
2. 提醒模式预设
3. 工作日 / 周末不同策略
4. 卡片展示偏好设置
5. 原因解释面板
6. 空状态和异常状态设计

### 9.3 P2 可后续迭代
1. 用户反馈闭环
2. 团队默认配置模板
3. 推荐算法优化
4. 配置导入导出

## 10. 关键需求
### 10.1 主页面
主页面更偏“设置控制台”而非“学习首页”。

主页面至少包含以下模块：
1. 今日状态
2. 当前提醒策略
3. 学习计划与静默时间
4. 卡片展示偏好
5. 系统与托盘设置

### 10.2 提醒策略
提醒策略分为两层：

1. 系统推荐层
- 根据产品默认策略生成推荐值
- 默认模式为“克制”
- 推荐值需要可读可解释

2. 用户覆盖层
- 用户可以直接修改推荐值
- 页面需显示当前为“推荐”还是“自定义”
- 用户可一键恢复为系统推荐

### 10.3 默认策略
默认值建议：
- idle 触发优先
- 空闲阈值偏高
- 夜间静默默认开启
- 每日新词上限保守
- 兜底触发保守
- 默认自动隐藏时长适中

### 10.4 托盘
托盘保留但降级为快捷工具，默认包含：
1. 打开主页面
2. 打开统计页
3. 暂停 1 小时
4. 今日不再提醒
5. 退出应用

### 10.5 浮卡
浮卡仍是最终交互组件，但不再承担产品主入口职责。

浮卡需支持：
- 当前单词展示
- 认识 / 不认识 / 跳过
- 自动隐藏
- 快捷键操作
- 与主页面设置联动

### 10.6 首次使用引导
首次打开应用时至少采集：
1. 每日新词目标
2. 可提醒时间段
3. 偏克制还是偏积极
4. 是否开机启动

## 11. 设置项清单
### 11.1 提醒策略
- 提醒模式：克制 / 平衡 / 强化 / 自定义
- idle 触发阈值
- 兜底提醒频率
- 是否启用兜底提醒

### 11.2 时间策略
- 夜间静默开始时间
- 夜间静默结束时间
- 工作日策略
- 周末策略

### 11.3 学习计划
- 每日新词上限
- 是否优先复习词
- 无复习词时是否引入新词

### 11.4 卡片偏好
- 自动隐藏时间
- 是否显示音标
- 默认展示顺序
- 是否允许跳过
- 快捷键开关

### 11.5 应用行为
- 开机启动
- 启动后显示主页面 / 最小化到托盘
- 托盘菜单开关

## 12. 关键页面与状态
### 12.1 主页面状态
- 首次使用
- 正常使用
- 已暂停
- 今日不再提醒
- 当前无可用卡片
- 配置已被用户覆盖

### 12.2 提醒状态
主页面需要告诉用户：
- 当前提醒模式
- 当前是否暂停
- 下次可能提醒时间
- 最近一次未提醒原因

## 13. 关键指标
### 13.1 产品指标
- 7 日留存
- 每日响应次数
- 跳过率
- 暂停率
- 用户对默认推荐的保留率
- 用户从推荐值切换到自定义的比例

### 13.2 可用性指标
- 启动可见率
- 主页面可理解性
- 设置完成率
- 空弹窗率
- 错误触发率

## 14. 风险与待确认项
1. 系统推荐逻辑第一版是否先用静态规则，不做动态学习
2. 主页面是单页控制台还是左侧导航多页结构
3. 工作日 / 周末策略是否纳入 P0
4. 自动隐藏时间是否允许关闭
5. 用户是否需要“会议中不提醒”等更复杂状态识别

## 15. 发布建议
v0.2 不应再以“托盘原型”对外讨论。

建议先完成以下最小集合再进入试点：
1. 主页面可见
2. 设置控制台可用
3. 推荐值与自定义覆盖生效
4. 托盘与主页面状态同步
5. 浮卡行为受配置驱动
