# 碎片时间背单词工具 MVP PRD

## 1. 文档信息
- 版本：MVP v0.1
- 状态：待立项
- 面向平台：macOS
- 技术路线：Tauri + TypeScript + SQLite
- 产品形态：菜单栏常驻应用 + 轻量浮卡

## 2. 项目背景
用户在日常工作中有大量 1-3 分钟的碎片空闲，但传统背词工具需要主动打开 App、进入学习页、开始任务，启动成本高，导致高频但短时的学习机会被浪费。

本项目希望把"背单词"从主动任务改成被动插入：系统在用户空闲时展示一张单词卡，用户只需 1 次点击或 1 个快捷键即可完成一次学习。

## 3. 产品目标
在 macOS 上验证以下假设：

1. 用户愿意在碎片时间完成 3 秒级的单词判断。
2. 如果打扰控制得当，用户不会因为高频弹出而关闭产品。
3. 即使只用极简复习算法，也能形成基本复学闭环。

## 4. 目标用户
- 备考雅思的桌面办公用户
- 有固定电脑工作时间的开发者、运营、内容、产品用户
- 希望利用工作间隙记词，而不是专门安排整段学习时间的人群

## 5. 核心价值
- 无需主动打开学习应用
- 每次学习成本极低
- 自动安排复习
- 让记词融入工作节奏，而不是与工作抢时间

## 6. MVP 成功标准
### 6.1 核心业务指标
1. 安装后 3 天仍保持开启的用户比例 >= 50%
2. 单词卡响应率 >= 40%
3. 跳过率 <= 35%
4. 每日平均完成学习次数 >= 10

### 6.2 产品可用性指标
1. idle 触发稳定，无明显漏弹或连续弹出
2. 回答后复习计划正确更新
3. 暂停和夜间免打扰可靠生效
4. 今日统计可与日志对账

## 7. MVP 范围
### 7.1 包含
1. 菜单栏常驻
2. 开机启动
3. 本地 SQLite 数据存储
4. 一套本地导入的雅思核心词库
5. 基于 idle 的卡片触发
6. 单词卡展示
7. `认识 / 不认识 / 跳过`
8. 快捷键操作
9. 固定复习间隔
10. 今日统计
11. 夜间免打扰
12. 暂停 1 小时

### 7.2 不包含
1. 编辑器保存触发
2. IDE 调试检测
3. 终端执行检测
4. CPU 占用检测
5. 成就系统
6. 连击系统
7. 晚间总结通知
8. 多设备同步
9. 复杂详情页
10. 例句、派生词、同反义词
11. 用户账号体系

## 8. 用户场景
### 场景 A：工作间隙快速学习
1. 用户停止操作电脑超过 90 秒
2. 应用检测到满足触发条件
3. 桌面右上角弹出 1 张单词卡
4. 用户按 `Cmd+K` 标记"认识"
5. 卡片消失，系统更新复习时间

### 场景 B：不会的词被继续追踪
1. 用户看到单词卡并点击"不认识"
2. 系统将该词回退到更早复习阶段
3. 该词在后续更短时间间隔内再次出现

### 场景 C：用户暂时不想被打扰
1. 用户从菜单栏点击"暂停 1 小时"
2. 应用在暂停期间不再弹卡
3. 到时自动恢复

## 9. 核心流程
1. 应用启动并常驻菜单栏
2. 初始化数据库和配置
3. 周期检测 idle 状态
4. 判断是否满足弹卡条件
5. 选取到期复习词，若无则选新词
6. 展示浮卡
7. 用户执行 `认识 / 不认识 / 跳过`
8. 更新复习计划、日志和今日统计

## 10. 交互设计
### 10.1 单词卡内容
展示字段：
- 英文单词
- 音标
- 词性
- 中文义

示例：

abandon
/əˈbændən/
v. 放弃；抛弃

[认识] [不认识] [跳过]

### 10.2 操作定义
- `认识`：本次回忆成功，进入下一复习阶段
- `不认识`：本次失败，回退一档
- `跳过`：本次不计入正确率，并进入短冷却
- `Esc`：等同跳过
- `Cmd+K`：认识
- `Cmd+J`：不认识

### 10.3 展示约束
1. 单次只显示 1 张卡
2. 默认展示 10-12 秒自动消失
3. 用户作答后立即消失
4. 未作答自动消失时，记为跳过

## 11. 触发策略
### 11.1 主触发
- 系统 idle >= 90 秒

### 11.2 兜底触发
- 距离上次展示 >= 25 分钟
- 且当前 idle >= 90 秒

### 11.3 弹卡前置条件
以下条件全部满足才可弹出：
1. 当前不在夜间静默时段
2. 当前不在手动暂停时段
3. 当前没有卡片显示中
4. 当前存在可展示的复习词或新词
5. 当前距离上次跳过冷却已结束

## 12. 免打扰规则
### 12.1 默认规则
- 夜间静默：23:00 - 07:00

### 12.2 用户控制
- 暂停 1 小时
- 今日不再提醒

### 12.3 首版可选规则
- 全屏应用时不弹
如果实现复杂，可延后到内测版。

## 13. 复习规则
采用固定复习间隔，而非复杂自适应算法。

### 13.1 阶段定义
- Stage 0：10 分钟
- Stage 1：1 天
- Stage 2：3 天
- Stage 3：7 天
- Stage 4：14 天

### 13.2 状态流转
- 新词首次答对后进入 Stage 0
- 每次答对前进一档
- 每次答错回退一档
- 已在 Stage 0 时答错，仍保持 Stage 0
- 在高阶阶段连续答对后可标记为 mastered

### 13.3 掌握定义
建议规则：
- 到达 Stage 4 后再次答对 1 次，标记为 `mastered`

## 14. 新词与复习配额
1. 每日新词默认上限：15
2. 展示优先级：到期复习词 > 新词
3. 当日新词配额用完后，只展示复习词
4. 若复习词积压，允许当天只做复习不引入新词

## 15. 菜单栏功能
菜单栏下拉面板包含：
1. 今日已学数量
2. 今日正确率
3. 今日新词数量
4. 待复习词数量
5. 暂停 1 小时
6. 今日不再提醒
7. 打开统计页
8. 退出应用

## 16. 统计页
MVP 仅展示数字，不做图表。

字段包括：
1. 今日展示次数
2. 今日认识次数
3. 今日不认识次数
4. 今日跳过次数
5. 今日新学词数
6. 待复习词数
7. 已掌握词数

## 17. 验收标准
### 功能验收
1. 应用可安装并开机自启
2. idle 条件满足时能弹出单词卡
3. 用户操作后数据正确写入 SQLite
4. 复习词能按到期时间再次出现
5. 暂停和静默时段生效
6. 统计页数据与日志一致

### 体验验收
1. 连续运行半天无重复弹卡和明显卡死
2. 不抢占主工作流焦点
3. 快捷键可靠可用
4. 弹卡频率可接受

## 18. 风险与待确认项
1. 全屏应用检测是否纳入 MVP
2. 词库来源及授权是否明确
3. 快捷键是否与用户现有系统快捷键冲突
4. 浮卡是否需要抢焦点才能接收快捷键
5. Tauri 在菜单栏场景下的稳定性是否满足预期
